## Prerequisites

<Callout kind="info" title="Before you begin">

Ensure you have the following ready:

- A Novisurf Storage account with access to your API key
- Node.js version 18 or later installed
- A code editor such as VS Code

</Callout>

## Install the Novisurf Storage SDK

Install the Novisurf Storage SDK from GitHub using your preferred package manager.

<CodeGroup tabs="npm,yarn,pnpm">
```bash
npm install github:yourusername/dreambase-sdk
```

```bash
yarn add github:yourusername/dreambase-sdk
```

```bash
pnpm add github:yourusername/dreambase-sdk
```
</CodeGroup>

## Configure environment variables

Define the endpoint and API key that the SDK uses to authenticate with Novisurf Storage.

Create a `.env` file in the root of your project:

```bash
DREAM_ENDPOINT="https://api.novisurf.top"
DREAMBASE_API_KEY="db_xxxxx_xxxxxxxxxxxxxxxxxx"
```

<Callout kind="tip">

Store `DREAMBASE_API_KEY` securely and never commit it to version control. Use environment managers or deployment platform secrets in production.

</Callout>

## Initialize the SDK

Create a minimal client that connects to Novisurf Storage using the environment variables.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
// src/storage.ts
import { Dreambase } from '@dreambase/sdk';

const storage = new Dreambase({
  endpoint: process.env.DREAM_ENDPOINT, // e.g., 'https://api.novisurf.top'
  apiKey: process.env.DREAMBASE_API_KEY,
});

export default storage;
```

```javascript
// src/storage.js
import { Dreambase } from '@dreambase/sdk';

const storage = new Dreambase({
  endpoint: process.env.DREAM_ENDPOINT, // e.g., 'https://api.novisurf.top'
  apiKey: process.env.DREAMBASE_API_KEY,
});

export default storage;
```
</CodeGroup>

If the configuration is correct, creating the `storage` instance does not throw an error and is ready to perform operations.

## Provision a bucket and upload your first object

Use a single project identifier (for example `myproject`) as the logical bucket key for all Novisurf Storage operations.

<Steps>
  <Step title="Provision a storage bucket" icon="plus">

Call `provisionStorageBucket` once for your project to create the underlying storage bucket and access keys.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
import storage from './storage';

async function setupBucket() {
  const result = await storage.provisionStorageBucket('myproject');

  if (!result.ok) {
    throw new Error('Failed to provision storage bucket');
  }

  console.log('Bucket created:', result.bucket);
  console.log('Access key:', result.accessKey);
}

setupBucket().catch(console.error);
```

```javascript
  console.log('Access key:', result.accessKey);
}

setupBucket().catch(console.error);
```
</CodeGroup>

On success, your terminal logs the new bucket name and access key, confirming that Novisurf Storage is ready for use.

  </Step>

  <Step title="Inspect your storage key and endpoint" icon="info">

Retrieve the storage key details to see which protocol and endpoint your bucket uses.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
import storage from './storage';

async function printStorageKey() {
  const key = await storage.getStorageKey('myproject');

  console.log('Protocol type:', key.protocolType); // 'ONPREM' or 'NF'
  console.log('Endpoint:', key.endpoint);
  console.log('Public domain:', key.domain); // e.g., 'storage.novisurf.top'
}

printStorageKey().catch(console.error);
```

</CodeGroup>

You should see `protocolType` as either `ONPREM` or `NF`, along with the resolved `endpoint` and `domain` for your bucket.

  </Step>

  <Step title="Upload an object" icon="upload">

Encode a small text file as base64 and upload it to your project bucket.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
import storage from './storage';

async function uploadHello() {
  const content = 'Hello, Novisurf Storage!';
  const contentBase64 = Buffer.from(content, 'utf8').toString('base64');

  await storage.uploadObject(
    'myproject',
    'hello.txt',
    contentBase64,
    'text/plain'
  );

  console.log('Uploaded hello.txt');
}

uploadHello().catch(console.error);
```

  </Step>

  <Step title="Make the bucket public and access the file" icon="globe-2">

Toggle the bucket to public and construct a public URL for `hello.txt`.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
import storage from './storage';

async function makePublicAndGetUrl() {
  // Set bucket to public
  await storage.setStorageBucketPublic('myproject', true);

  // Get storage key to retrieve bucket name and domain
  const key = await storage.getStorageKey('myproject');
  
  // Construct public URL
  const publicUrl = `https://${key.domain}/${key.bucket}/hello.txt`;

  console.log('Public URL:', publicUrl);
}

makePublicAndGetUrl().catch(console.error);
```

The public file URL follows the pattern `https://storage.novisurf.top/{bucket}/hello.txt`. Open the printed URL in a browser to see the file contents.

  </Step>

  <Step title="Delete an object and check quota" icon="trash">

Clean up the demo file and inspect your current storage usage and operation counts.

<CodeGroup tabs="TypeScript,JavaScript">
```ts
import storage from './storage';

async function cleanupAndQuota() {
  await storage.deleteObject('myproject', 'hello.txt');
  console.log('Deleted hello.txt');

  const quota = await storage.getQuota('myproject');

  console.log('Storage used:', quota.storage?.used, 'bytes');
  console.log('Storage limit:', quota.storage?.limit, 'bytes');
  console.log('Read ops:', quota.operations?.reads);
  console.log('Write ops:', quota.operations?.writes);
}

cleanupAndQuota().catch(console.error);
```

## Rate limits and best practices

Novisurf Storage applies per-project rate limits of approximately 20 write operations per minute and 50 read operations per minute.

<Callout kind="info">

Batch non-critical writes where possible, and cache frequently accessed objects on the client or behind your own CDN to stay within read limits.

</Callout>

<Expandable title="Troubleshooting common issues" default-open="false">

- **Unauthorized or 401 errors**: Confirm `DREAMBASE_API_KEY` is valid and loaded into your Node.js process.
- **Missing environment variables**: Ensure `DREAM_ENDPOINT` and `DREAMBASE_API_KEY` are defined in `.env` and that you call `dotenv.config` before initializing the SDK.
- **Object not found at public URL**: Check that you called `setStorageBucketPublic('myproject', true)` and that the path matches `https://storage.novisurf.top/{bucket}/{filename}`.
- **Rate limit exceeded**: Space out bulk uploads and add client-side or CDN caching to reduce read traffic.

</Expandable>
